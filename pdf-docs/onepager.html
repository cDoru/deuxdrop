Deuxdrop: Privacy Focused Messaging

The benefits of the cloud without losing control.

Key Features:

  1. Messages live in the cloud:
  1. redundantly backed up
  2. accessible from your computer, your tablet, your phone
  1. Server smarts:
  1. multi-device aware: only acknowledge a new message once
  2. device subscriptions: not all devices need to know about everything
  1. Strong identity means you only receive from people you want to receive from.
  2. Conversation-centric.  New participants get a first class copy of the backlog.
  3. Federated: anyone can run their own server.
  4. Encryption keeps your messages private.

Implementation:

  1. All program logic is written in JavaScript, allowing reuse of code between client and server and allowing functionality migration.
  2. Clients built on web technologies: HTML, JS, IndexedDB, WebSockets.
  3. A small JS API makes it feasible/easy to build new UIs without reimplementing everything.
  4. Crypto currently provided by a native library (djb’s nacl), but we can switch to the W3C standard being built around Mozilla’s DOMCrypto work once it stabilizes.

Relationship to existing protocols / technologies:

  1. XMPP/Jabber.  XMPP can be thought of as a transport layer with popular extensions supporting instant messaging and chat semantics.  There are no extensions providing message storage and access semantics like those provided by gmail or even IMAP.  XMPP could be adopted as the transport layer for inter-server communication.  Currently, inter-server communication is accomplished via reuse of the same protocol between the user’s client and their mail store, simplifying the implementation.
  1. IMAP.  Deuxdrop currently does not speak IMAP.  IMAP is a complex beast with many extensions, all of which have varying levels of support and adoption.  The major reasons for avoiding it are as follows:
  1. Poorly aligned with conversation-centric representations.  IMAP is built around folders.
  2. Server load concerns.  It is easy for clients to generate significant server load.
  3. Privacy issues.  The standard idiom leaves servers with authoritative control over all metadata and the addition of new messages.  Additionally, even PGP-encrypted e-mail leaves servers with access to all headers, message subjects, etc.
  1. SMTP.  The traditional world of e-mail, with weak identity, limited support for conversations, and unencrypted messages by default.  We will create a gateway to provide inter-operation so that Deuxdrop is not its own isolated corner of the universe.
